## 二、今日修复的问题

### 2.1 数据库嵌套目录问题

**问题描述**:
- 发现存在嵌套目录 `lib/db/lib/db/` 包含实际的数据库文件
- `lib/db/test.db` (880KB) 和 `lib/db/lib/db/test.db` (1093KB) 内容不同
- `.env` 配置的路径指向空文件 `lib/db/dev.db`

**解决方案**:
1. 删除嵌套目录 `lib/db/lib/`
2. 用正确数据替换数据库文件
3. 更新 `.env` 配置指向正确路径

**文件结构修复后**:
```
lib/db/
├── dev.db       (995KB - 有数据)
├── test.db      (1093KB - 有数据)
├── schema.prisma
└── migrations/
```

### 2.2 Prisma数据库路径解析问题

**问题描述**:
- `file:./lib/db/test.db` 是相对路径，在Next.js运行时解析错误
- 导致Prisma客户端无法正确打开数据库

**解决方案**:
修改 `lib/db/prisma.ts`，使用绝对路径:

```typescript
function getDatabaseUrl(): string {
  const dbPath = process.env.DATABASE_URL || 'file:./lib/db/dev.db';
  if (dbPath.startsWith('file:')) {
    const dbFile = dbPath.slice(5);
    if (!path.isAbsolute(dbFile)) {
      return 'file:' + path.resolve(process.cwd(), dbFile);
    }
  }
  return dbPath;
}
```

### 2.3 标签筛选功能 - 状态更新异步问题

**问题描述**:
- 每次点击标签，题目数量显示不同（8道、125道等）
- 原因: `setSelectedTagIds()` 是异步的，但 `fetchQuestions()` 同步调用，使用了旧状态

**解决方案**:
1. 添加 `useEffect` 监听状态变化自动刷新:

```typescript
useEffect(() => {
  fetchQuestions();
}, [filter.status, filter.grade, filter.type, search, selectedTagId, selectedTagIds]);
```

2. 移除各处直接调用 `fetchQuestions()` 的代码，避免重复请求
