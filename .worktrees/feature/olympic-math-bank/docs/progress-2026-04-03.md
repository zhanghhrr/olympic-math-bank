# 项目进展 2026-04-03

## 1. 答案清洗功能优化

### 问题
OCR识别后的答案包含【标注】、【业务题型】等无关文本，需要清洗。

### 解决方案
修改 `lib/ocr/question-identifier.ts`：

1. **新增 `removeUselessAnswerTags` 函数**（第891-920行）
   - 移除 `![](images/...)` 图片标记
   - 移除 `【标注】`、`【业务题型】`、`【知识点】`、`【能力】`、`【思想】` 等无关标签

2. **修复解析提取正则**（第1039行）
   - 从 `(?=\n\n|$)` 改为 `(?=【|$)`，正确提取【解析】到下一个【标签之前的内容

3. **修复选择题检测逻辑**（第777-781行）
   - 原来 `[A-D][\.、\s]` 会误匹配单词中的字母（如"A B"中的"A "）
   - 现在使用更严格的正则，只匹配真正的选项格式

4. **移除题型相关的答案精简逻辑**
   - 用户明确：答案就是【答案】之后、【解析】之前的文本，不根据题型进一步精简

### 验证结果
- 20道题目全部识别成功
- 所有答案都干净，不含【标注】、【业务题型】等无关文本
- 选择题答案正确（如"C"）
- 填空题答案完整（如"（1）90. (2)54"）

## 2. 导入页面增强功能

### 需求
在OCR识别完成后、确认导入前，增加：
- 手动打标签功能（批量选择题目 + 知识标签树）
- 手动修改题干/答案/解析功能
- 确认后进入待审核状态

### 用户确认
- 批量打标签：替换现有标签
- 不需要批量删除功能
- 需要二次确认：先预览，确认后改为PENDING

### 实现方案

#### Phase 1: 新增批量更新API
**文件**: `app/api/questions/batch/route.ts`
- PATCH: 批量更新题目内容、答案、解析、标签、状态
- DELETE: 批量删除题目
- 替换模式更新知识标签

#### Phase 2: 新增确认导入API
**文件**: `app/api/import/confirm/route.ts`
- 接收前端传来的题目数据
- 自动匹配知识标签
- 创建题目，状态设为PENDING

#### Phase 3: 重构OCR API
**文件**: `app/api/import/ocr/route.ts`
- OCR识别后返回预览列表，不写入数据库
- 返回临时ID供前端编辑使用

#### Phase 4: 新增可编辑题目卡片
**文件**: `components/question/question-card.tsx`
- Checkbox批量选择
- 点击展开可编辑题干、答案、解析
- 显示/删除当前标签

#### Phase 5: 新增批量打标签弹窗
**文件**: `components/knowledge-tag/batch-dialog.tsx`
- 知识标签树选择
- 替换模式批量打标签

#### Phase 6: 重构导入页面
**文件**: `app/dashboard/import/page.tsx`
- 预览编辑模式
- 批量操作工具栏（显示选中数量、批量打标签按钮）
- 确认导入流程

## 3. 已知问题与修复

### 问题1: 数据库无法打开
**错误**: `Error code 14: Unable to open the database file`

**原因**: `.env` 中数据库路径是相对路径 `file:./lib/db/test.db`，在某些情况下无法正确解析。

**解决**: 将 `.env` 中的 `DATABASE_URL` 改为绝对路径：
```
DATABASE_URL="file:C:/Users/Twilight/.codebuddy/.worktrees/feature/olympic-math-bank/lib/db/test.db"
```

### 问题2: 函数未导出
**错误**: `Export autoMatchKnowledgeTags doesn't exist in target module`

**原因**: 辅助函数 `autoMatchKnowledgeTags`、`detectQuestionType` 等未导出。

**解决**: 在 `lib/ocr/import-to-db.ts` 中将这些函数改为 `export`：
- `export async function autoMatchKnowledgeTags(...)`
- `export function detectQuestionType(...)`
- `export function estimateDifficulty(...)`
- `export function stripQuestionNumber(...)`
