# 🌟 奥数题库系统开发进展（2026-05-26）

## 1. 🏷️ 题型识别增强

针对 OCR 导入后的题目，实现了完整的四类题型自动检测系统（位于 `lib/ocr/question-identifier.ts`）：

| 题型 | 检测策略 |
|------|----------|
| **选择题** | ≥3 行 A/B/C/D 行首前缀、行内连续选项 `A. B. C. D.`、严格上下文去重计数 ≥3、关键词「选择/选项」 |
| **填空题** | 关键词（填空/填入/填写）、空白占位符（`_{2,}` / `□` / `（）`）、LaTeX `\underline`、**结构推断**（OCR 丢失下划线时通过语义自动识别） |
| **计算题** | 关键词「计算/口算/脱式/竖式/递等式」 |
| **解答题** | 兜底策略（不匹配以上三类的题目） |

- 新增 `HybridQuestionIdentifier.questionTypeToDB()` 静态方法，将中文题型（填空题/选择题/解答题/计算题）映射为 Prisma 数据库枚举（`FILL_BLANK` / `CHOICE` / `SOLUTION` / `CALCULATION`）。
- 消除了 `question-identifier.ts` 与 `import-to-db.ts` 中重复的 `detectQuestionType`，统一为单一来源。

## 2. ✍️ 填空下划线恢复 —— 核心难题

**问题背景**：PDF 中印刷的填空下划线通常只有约 1px 粗细，RapidOCR 的 PP-OCRv4 识别模型无法将如此细的水平线识别为 `_` 字符，导致输出文本中留下空格，填空题被误判为解答题。

**解决方案**（双层次处理）：

### 2.1 Python 端：图片预处理水平线增强（`lib/ocr/pdf2md_rapidocr.py`）
- 新增 `enhance_underlines()` 预处理函数
- 原理：垂直模糊差减法 —— 对灰度图做垂直方向模糊得到"不含水平线"的参考图，原图减参考图突出水平细线区域，再将检测到的细线上下各扩展 1px 加粗
- 默认 DPI 从 200 提升到 400，提高细节分辨率
- 支持 `--no-enhance` 参数跳过预处理（调试用）

### 2.2 TypeScript 端：后处理插入占位符（`lib/ocr/question-identifier.ts`）
- `inferBlankPlaceholders()` 方法：检测「语义期待数值的词 + 空格 + 量词」模式，自动插入 `______`
- 触发词：`相距`、`是`、`为`、`需要`、`共`、`长/宽/高` 等
- 量词：`米`、`千米`、`米/分`、`分钟`、`元`、`个` 等
- `detectQuestionType` 结构推断放宽句末标点判定：原先只接受 `。.` 结尾，现在也接受 `，,；;`
- 效果验证：`"相距 米。"` → `"相距______米。"` ； `"是 米/分，"` → `"是______米/分，"` ✅

## 4. 🐍 Python OCR 脚本 —— 全新独立模块

### 4.1 `lib/ocr/pdf2md_rapidocr.py`（352 行，全新文件）
将 PDF 转换从 MinerU 依赖中解耦，建立独立的 RapidOCR 本地链路：

- PyMuPDF 渲染 PDF → PNG 图片（可配置 DPI）
- Docling 版面分析 + RapidOCR (PP-OCRv4 mobile) 进行 OCR 识别
- 内置水平线增强预处理
- 支持输出 Markdown 和 JSON 格式
- 自动清理中间图片

### 4.2 `lib/ocr/rapidocr-client.ts`（324 行，全新文件）
TypeScript 端调用封装：
- 通过 `child_process` 调用 Python 脚本
- 支持超时控制、错误处理
- 返回结构化 OCR 结果
