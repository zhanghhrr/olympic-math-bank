# 🏔️ 奥数题库管理系统 — 里程碑规划

> 项目代号：Olympic Math Bank\
> 所属公司：深圳桃李未来\
> 定位：面向数学竞赛培训的智能化题库生产、管理与组卷平台

***

## 项目愿景

打造教育行业领先的**数学竞赛题库数字化基础设施**，实现从纸质教材到结构化题库的全自动化流水线，从根本上解决传统教培机构"有题无库、有库无标、有标难用"的痛点。最终形成覆盖小学全年级、全竞赛层级的知识体系数据库，为桃李未来的教学教研提供数据驱动的决策支撑。

***

## 里程碑路线图

```
Phase 1 (已完成)     Phase 2 (进行中)     Phase 3 (规划中)      Phase 4 (远期)
 基础框架搭建    →    OCR自动化导入    →    智能组卷引擎     →   开放教学平台
     ✅                  🔄                     📋                    💡
```

## Phase 1：基础框架搭建 ✅ 已完成

**目标：** 建立可用的题库管理基本链路。

### 已交付成果

| 模块           | 功能                                              |
| ------------ | ----------------------------------------------- |
| **数据库模型**    | Prisma + SQLite，完整的题目、标签、知识体系、审核流 Schema        |
| **用户与权限**    | 三级角色体系（Admin / Reviewer / Editor），NextAuth 登录鉴权 |
| **题目 CRUD**  | 题目创建、编辑、删除，支持四类题型（填空/选择/解答/计算）                  |
| **五级知识标签**   | 模块→专题→子专题→知识点→技能，支持树状层级与编码索引                    |
| **审核工作流**    | Draft → Pending → Approved / Rejected，含审核记录     |
| **版本管理**     | 题目修改历史追踪，版本快照与回滚（content / answer / formulas）   |
| **PDF 组卷打印** | Notion-like 块状排版引擎，拖拽排序，手动分页，学生/教师双模式           |
| **标签管理**     | 年级/难度/知识点/竞赛类型四类标签系统                            |

### 关键指标

- 数据库模型完整度：14 张表
- 支持年级：P1-P6（小学 1-6 年级）
- 支持题型：4 种
- 知识标签层级：5 级

## Phase 2：OCR 自动化导入 🔄 进行中

**目标：** 实现纸质 PDF 教材 → 结构化题库的端到端自动化。

### 已交付

| 模块                | 功能                                    |
| ----------------- | ------------------------------------- |
| **RapidOCR 本地链路** | 独立 Python 脚本，PDF→图片→OCR→Markdown/JSON |
| **水平线增强**         | 垂直模糊差减法预处理，解决 OCR 下划线识别丢失问题           |
| **题型自动识别**        | 三层策略（关键词/占位符/结构推断），四类题型精确分类           |
| **填空下划线恢复**       | 语义推断 + 后处理插入占位符                       |
| **内容清洗流水线**       | 9 步处理：去标题残留、断行合并、中文空格清理、数字残留修复        |
| **答案提取**          | 答案/解析/标注分离，无关标签自动过滤                   |
| **自动标签匹配**        | `autoMatchKnowledgeTags`，基于五级知识体系自动打标 |
| **导入任务追踪**        | ImportJob / ImportJobItem，全流程状态监控     |

### 待交付

| 模块             | 说明                                        |
| -------------- | ----------------------------------------- |
| **批量导入**       | 批量上传 PDF 文件夹，一键全量导入，进度可视化                 |
| **OCR 精度评测**   | 建立标准评测集，追踪识别准确率趋势                         |
| **公式验证增强**     | 更多 LaTeX 公式的自动比对与纠错逻辑                     |
| **多 OCR 引擎切换** | 支持 RapidOCR / MinerU / SimpleTex 多引擎配置与降级 |
| **图片题识别**      | 几何图形题的图文分离与结构保留                           |

## Phase 3：智能组卷引擎 📋 规划中

**目标：** 教师一键生成难度适配、知识点覆盖的高质量试卷。

### 核心能力

| 功能          | 描述                             |
| ----------- | ------------------------------ |
| **智能选题**    | 根据目标年级、难度分布、知识点覆盖要求，自动筛选题目     |
| **难度均衡算法**  | 确保试卷整体难度符合目标曲线，避免偏难或偏易         |
| **知识点覆盖分析** | 可视化展示试卷对各知识模块的覆盖比例，识别薄弱环节      |
| **题型自动配比**  | 按预设模板自动分配填空/选择/解答/计算题型比例       |
| **历史去重**    | 与历史考卷比对，避免近期重复出题               |
| **组卷建议**    | 基于历年竞赛真题数据，推荐高频考点与题型组合         |
| **一键导出**    | 生成标准 A4 试卷 PDF + 答题卡 + 教师版答案解析 |

### 业务价值

- 教师组卷时间从 **2 小时 → 15 分钟**
- 题库利用率从 <10% → >60%
- 避免人为偏好导致的考点遗漏

## Phase 4：开放教学平台 💡 远期愿景

**目标：** 从内部工具成长为面向教师、学生、家长的多端教学服务平台。

### 蓝图

| 模块           | 功能                        |
| ------------ | ------------------------- |
| **学生端**      | 个性化练习推送、错题本、进度追踪、能力雷达图    |
| **教师端**      | 学情看板、批量组卷、作业批改辅助          |
| **家长端**      | 学习报告、阶段性成长曲线、薄弱点预警        |
| **竞赛备考系统**   | 历年真题库、模拟考试、排名评估           |
| **API 开放平台** | 对外提供题库检索、OCR 识别、自动组卷 API  |
| **数据中台**     | 各校区使用数据汇聚，教研趋势分析，题目质量自动评估 |
| **AI 助教**    | 基于 LLM 的解题思路生成、错误分析、个性化讲解 |

### 业务价值

- 标准化教学输出，降低对名师个人依赖
- 各校区教学质量一致性保障
- 形成数据资产壁垒，构建核心竞争力

***

## 预期效益总览

| 维度       | 当前痛点               | 预期改善                |
| -------- | ------------------ | ------------------- |
| **题目生产** | 手动录入，人均 10 题/小时    | OCR 自动导入，≥100 题/小时  |
| **题库规模** | 零散 Excel 管理，<500 题 | 结构化数据库，目标 10,000+ 题 |
| **组卷效率** | 人工拼凑，2 小时/份        | 智能组卷，15 分钟/份        |
| **教研数据** | 凭经验判断，无数据支撑        | 全维度数据看板，精准教研决策      |
| **校区协同** | 各校区独立出题，质量参差       | 统一题库 + 审核流，品质一致     |
| **知识沉淀** | 教师离职带走经验           | 题库 + 标签 + 解析三位一体持久化 |
