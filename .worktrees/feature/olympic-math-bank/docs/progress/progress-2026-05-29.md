# 🌟 奥数题库系统开发进展（2026-05-29）

基于 [docs/workflow-optimization.md](file:///c:/Users/Twilight/.codebuddy/.worktrees/feature/olympic-math-bank/docs/workflow-optimization.md) 的设计方案，按优先级顺序实施了全部 10 项优化。

## P0 · OCR 结果缓存 🚀

- **文件**：`lib/ocr/mineru-client.ts`
- **方案**：以 PDF 文件 SHA256 为 key，缓存 OCR 结果（`full.md` + `_content_list.json`）到 `uploads/ocr/.cache/`
- **效果**：重复导入同一 PDF 时跳过 MinerU API 调用，命中后秒级返回
- **实现**：新增 `computeFileHash()`、`readCache()`、`writeCache()` 三个辅助函数，在 `processPDF()` 入口先检查缓存

## P1 · 标签置信度色标展示 🎨

- **文件**：`lib/ocr/tagging.ts`、`app/api/import/ocr/route.ts`、`components/question/question-card.tsx`
- **方案**：将 `calculateTagScore()` 计算出的匹配分数透传到前端，以绿/黄/红三色标记置信度
  - 🟢 绿色（score ≥ 5）：章节标题/标注文本精确匹配
  - 🟡 黄色（score ≥ 3）：关键词匹配
  - 🔴 红色（score < 3）：LLM 推断
- **新增**：`autoMatchKnowledgeTagsWithScores()` 函数，返回 `{ tagIds, scoredTags }` 含分数与来源
- **辅助函数**：`getConfidenceLevel()`、`getMatchSourceLabel()` 色标渲染

## P2 · 批量编辑工具栏 ✏️

- **文件**：`app/api/questions/batch/route.ts`、`app/dashboard/questions/page.tsx`
- **方案**：在题目列表"组卷购物车"中增加批量操作能力
  - "批量编辑"按钮 → 展开年级/难度下拉选择器 → 调用 `PATCH /api/questions/batch`
  - "批量打标签"按钮 → 复用 `BatchTagDialog` 组件
- **API 增强**：batch PATCH 新增 `grade` 字段支持
- **状态管理**：`handleBatchUpdate()`、`handleBatchTagConfirm()` 含 loading 和错误处理

## P3 · PDF/文本对照视图 📄

- **文件**：`app/api/import/ocr/route.ts`、`app/dashboard/import/page.tsx`、`components/question/question-card.tsx`
- **方案**：导入预览页改为左右分栏布局
  - 左侧：`<iframe>` 嵌入原始 PDF（浏览器内置阅读器）
  - 右侧：题目卡片列表
  - 每道题卡片显示所在页码 badge
- **后端**：OCR API 新增 `pdfUrl` 返回 + 从 `sourceBlocks` 提取每道题的 `pages[]`
- **新增字段**：`QuestionPreview.pages`、页面 badge 渲染

## P4 · 异步导入任务机制 ⚡

- **文件**：`app/api/import/ocr/async/route.ts`（全新）、`app/dashboard/import/page.tsx`
- **方案**：OCR 处理改为异步任务模型
  - `POST /api/import/ocr/async` → 创建 `ImportJob`，立即返回 `jobId`
  - 后台 `processJobAsync()` 调用 MinerU → 更新 ImportJob 状态
  - 前端 `setInterval` 每 2 秒轮询 `GET /api/import/ocr/async?jobId=X`
- **前端开关**：新增"异步导入"复选框（默认开启），关闭可走同步路径
- **超时保护**：最大轮询 300 次（10 分钟）

## P5 · 题目去重 🔍

- **文件**：`lib/ocr/dedup.ts`（全新）、`app/api/import/confirm/route.ts`
- **方案**：基于 SimHash 文本指纹的去重模块
  - `computeSimHash()`：字符 trigram → MD5 hash → 64 位投票 → 指纹
  - `checkDuplicates()`：单题检查
  - `batchCheckDuplicates()`：批量检查（复用数据库查询）
- **入库集成**：`confirm` API 在批量创建前执行去重，相似度 > 95% 自动跳过
- **返回信息**：标注重复题的相似度和已有题目 ID

## P6 · 多格式导出 🗂️

- **文件**：`app/api/export/json/route.ts`（全新）、`app/api/export/csv/route.ts`（全新）、`app/dashboard/questions/page.tsx`
- **方案**：在题目列表购物车中增加一键导出功能
  - JSON 导出：完整结构化数据（含标签五级路径、公式、创建信息）
  - CSV 导出：BOM UTF-8 编码，Excel 友好，含 14 列字段
  - MD 导出（已有，新增按钮入口）
  - PDF 试卷（已有 `print/page.tsx`，保持不变）
- **前端**：`handleExport(format)` 统一处理下载（Blob → URL → a 标签下载）
- **新增路由**：`/api/export/json`、`/api/export/csv`

## P7 · 双引擎公式交叉验证 🔬

- **文件**：`lib/ocr/dual-verifier.ts`（全新）、`lib/ocr/formula-verifier.ts`
- **方案**：双引擎 LaTeX 对比框架
  - `compareFormulas()`：归一化 → Levenshtein 距离 → 一致性评分
  - `batchCompareFormulas()`：批量对比 MinerU vs SimpleTex
  - `generateDiffDetail()`：差异诊断（分数/根号/求和符号数量对比）
  - `isSimpletexAvailable()`：检测 SimpleTex API 是否配置
- **集成**：`verifyFormulas()` 新增可选参数 `simpletexFormulas[]`，自动触发双引擎验证

## P8 · 统计分析 API 📊

- **文件**：`app/api/stats/route.ts`（全新）
- **查询维度**（8 个并行查询）：
  - 题目总览（按状态分：草稿/待审/已通过/已驳回）
  - 题目分布（按年级/题型/难度）
  - 标签覆盖率（已用标签 vs 总标签数，Top 20 高频标签）
  - 导入趋势（最近 50 条 ImportJob）
  - 审核统计（按审核状态分）
- **新增路由**：`/api/stats`

## P9 · 编辑版本历史 📝

- **文件**：`app/api/questions/[id]/route.ts`、`app/api/questions/[id]/versions/route.ts`（全新）
- **方案**：每次编辑自动保存版本快照
  - PUT 路由：更新前读取当前题目 → `QuestionVersion.create()` 保存旧版 → 再执行更新
  - 自动生成 `changeLog`（检测题干/答案/解析变更）
- **版本管理 API**：
  - `GET /api/questions/[id]/versions`：列出所有历史版本
  - `POST /api/questions/[id]/versions`：回滚到指定版本（先保存当前状态为新版本，再恢复目标版本）
- **新增路由**：`/api/questions/[id]/versions`
