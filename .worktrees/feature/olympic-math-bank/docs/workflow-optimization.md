# 题库工作流优化空间分析

> 生成日期：2026-05-29 | 基于当前全链路审视

---

## 当前流水线

```
PDF 上传 → MinerU OCR → 题目拆分 → 自动打标签 → 年级推断/难度预估 → 预览确认 → 入库 → 导出
```

---

## 🔴 高优先（投入产出比最高）

### 1. 异步导入任务机制

**现状**：OCR 在 API Route 中同步等待。多页 PDF（30 页+）MinerU 可能耗时 2-5 分钟，Next.js API Route 有超时风险。

**方案**：改为异步任务模型——上传 PDF → 立即返回 `importJobId` → 后台轮询 MinerU → 前端每秒 poll 状态。

**已有基础**：`ImportJob` 表已存在，但未充分利用其状态机。

### 2. 导入预览页的 PDF/文本对照视图

**现状**：预览页只展示纯文本，无法和原 PDF 对照核对。公式密集的奥数题人工校验效率低。

**方案**：左侧 PDF 缩略图 + 右侧识别结果（Markdown 渲染 LaTeX），支持逐题高亮跳转。

**已有基础**：`formulas` 和 `sourceBlocks` 字段已提供定位信息（bbox, page），可直接利用。

### 3. 题目去重检测

**现状**：重复导入同一 PDF 会创建完全重复的题目，无任何去重逻辑。

**方案**：基于题干文本的 SimHash / MinHash 指纹入库前检查，相似度 > 95% 则提示"疑似重复，跳过/覆盖/保留两份"。

### 4. OCR 结果缓存

**现状**：开发调试阶段反复测试同一 PDF 的 OCR，每次都重新调用 MinerU API（消耗 token + 时间）。

**方案**：以 PDF 文件 SHA256 为 key，缓存 `_content_list.json` 和 `full.md` 到 `uploads/ocr/.cache/`，命中则跳过 API 调用。

---

## 🟡 中优先（显著提升体验）

### 5. 批量编辑能力

**现状**：题目列表页只能逐条编辑。14 道题统一改年级或标签需操作 14 次。

**方案**：复选框多选 → 批量修改年级 / 难度 / 标签。

**实施要点**：
- 前端：题目列表添加 checkbox + 批量操作工具栏
- 后端：`PATCH /api/questions/batch` 端点，接收 `{ ids, updates }`

### 6. 标签置信度可视化

**现状**：`matchedTags` 返回时无置信度分数，用户看不出哪些是"高确信"、哪些是"猜测的"。

**方案**：导入预览页展示每个标签的匹配分数和来源（章节标题 / 内容关键词 / LLM），用颜色区分：
- 🟢 绿色：高置信（章节标题映射 ≥ +5）
- 🟡 黄色：中置信（内容关键词匹配 ≥ +3）
- 🔴 红色：低置信（LLM 兜底匹配或低分）

**已有基础**：`calculateTagScore()` 已在 `tagging.ts` 中实现评分逻辑，只需透传分数到前端。

### 7. 多格式导出

**现状**：仅支持 MD 导出。

**实际需求**：
- 教师需要 **PDF 试卷**（带排版、留空）
- 教研需要 **DOCX**（可编辑）
- 数据分析需要 **JSON/CSV**

**已有基础**：打印排版模块（`DocumentBlock` + CSS `@media print`）已完成，可直接转 PDF。

---

## 🟢 低优先（长期迭代）

### 8. 双引擎交叉验证

**现状**：仅 MinerU 单一引擎。

**方案**：接入 SimpleTex 作为公式级别校验引擎——对比两个引擎的 LaTeX 输出，标注差异行，降低公式识别错误。

### 9. 题库统计分析仪表盘

按年级/模块/难度维度的题目数量分布、标签覆盖率趋势、导入量趋势。

### 10. 编辑版本历史

题目被修改后无审计记录。可用 `QuestionVersion` 表记录每次编辑的 diff，支持回滚。

## 投入产出矩阵

```
                高影响
                  │
      ① 异步导入   │   ② PDF/文本对照
      ③ 去重检测   │   ④ OCR 缓存
                  │
  ────────────────┼────────────────
                  │
      ⑤ 批量编辑   │   ⑥ 标签置信度
      ⑦ 多格式导出 │   ⑧ 双引擎验证
                  │
      ⑩ 版本历史   │   ⑨ 统计分析
                  │
                低影响
    低投入 ←──────────→ 高投入
```

## 建议执行顺序

| 优先级 | 优化项 | 预估工作量 | 投入产出比理由 |
|--------|--------|-----------|---------------|
| **P0** | OCR 缓存 | 改 ~10 行 | 立竿见影省 API 调用，调试效率翻倍 |
| **P1** | 标签置信度展示 | 前端色标 | 不改后端，用户审核效率大幅提升 |
| **P2** | 批量编辑 | API + 前端 | 高频操作，14 次 → 1 次 |
| **P3** | PDF/文本对照视图 | 前端为主 | 人工审核环节最关键的能力 |
| **P4** | 异步导入 | 后端重构 | 解决生产环境超时，但改动面大 |
| **P5** | 题目去重 | 指纹 + 检查 | 数据质量保障 |
| **P6** | 多格式导出 | 扩展现有模块 | 复用已有排版模块 |
| **P7** | 双引擎验证 | API 对接 | 需额外 API 接入 |
| **P8** | 统计分析仪表盘 | 全栈 | 锦上添花 |
| **P9** | 编辑版本历史 | DB + 前端 | 锦上添花 |
