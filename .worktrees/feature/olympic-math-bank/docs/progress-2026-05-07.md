## 一、修复问题汇总

### 1.1 图片对齐/尺寸保存后不显示（题目管理页 & 预览模式）

**问题描述**：
- 在编辑模式下调整图片尺寸/对齐后保存，在题目管理页和预览模式中图片全部显示为左对齐，尺寸也失效。

**根本原因**：
1. `components/QuestionContent.tsx` 第191-197行：处理对齐块（`:::left ... :::`）时，把内部所有 part 的 `type` 强制覆盖为 `'align'"`，导致图片的 `type: 'image'` 信息丢失，渲染时直接把 Markdown 文本插入 `innerHTML`，图片无法显示。
2. 对齐块正则只匹配 `\n`（LF），而 Windows 环境下保存的内容含有 `\r\n`（CRLF），导致正则匹配失败，`align` 属性始终为 `undefined`，图片全部退化为左对齐。
3. 渲染时使用 `text-align` 对齐，对 `display:block` 的图片无效。

**修复方案**：
- `QuestionContent.tsx` 和 `VisualLatexEditor.tsx` 中对齐正则 `\n` 改为 `\r?\n`，兼容 Windows CRLF
- `QuestionContent.tsx` 保留原始 `type`，只添加 `align` 属性；渲染时先按 `type` 正常渲染，再用 `flexbox` 布局实现对齐
- `VisualLatexEditor.tsx` 预览渲染也改为 `flexbox`（`justify-content: center/flex-end`）

**修改文件**：
| 文件 | 修改内容 |
|------|----------|
| `components/QuestionContent.tsx` | 修复对齐块解析（`\r?\n`）、修复渲染逻辑（`flexbox` 替代 `text-align`） |
| `components/VisualLatexEditor.tsx` | 修复对齐正则、修复预览模式渲染（`flexbox`） |

### 1.2 Turbopack 崩溃导致开发服务器无法启动

**问题描述**：
- 运行 `npm run dev:test` 启动 Turbopack 开发服务器时，报 `An unexpected Turbopack error occurred`，panic 日志显示处理 `app/globals.css` 时 Node.js 子进程崩溃（`exit code: 0xc0000142` = Windows DLL 初始化失败）。

**根本原因**：
- Turbopack 处理 `globals.css` 中的 Tailwind `@theme` 语法时需要启动 Node.js 子进程运行 PostCSS，在 Windows 上子进程启动失败。

**修复方案**：
- 改用 `next build && next start` 生产模式启动，绕过 Turbopack 的 Windows 兼容问题
- 在 `next.config.ts` 中无需特殊配置（`turbopack: false` 不是有效配置，已还原）

## 二、优化功能

### 2.1 自动打标签逻辑优化

**背景**：
- 数据库中有 **1473 个知识标签**，但 `lib/ocr/knowledge-keywords.ts` 只有 **~170 条**显式关键词
- `autoMatchKnowledgeTags()` 只用了这 ~170 条关键词，剩余 1300+ 个标签完全无法被匹配到
- 原来只返回**最匹配的 1 个标签 + 父级**，一次导入最多打 5 个标签，覆盖率极低

**优化方案**（`lib/ocr/import-to-db.ts`）：

1. **利用全部 1473 个标签**：为每个标签自动构建搜索词 = `标签名 + 所有父级标签名 + 显式关键词库`，覆盖率从 ~12% 提升到接近 100%
2. **返回前 5 个最匹配标签**（原来只返回 1 个），每个标签携带其父级 ID，一次导入可打 5~25 个标签
3. **中文精确匹配优化**：含 CJK 字符的关键词用标点/空格边界检测（`\b` 对中文无效）

**修改文件**：
| 文件 | 修改内容 |
|------|----------|
| `lib/ocr/import-to-db.ts` | 重写 `autoMatchKnowledgeTags()`，新增 `getParentNames()` 辅助函数 |
| `app/api/import/confirm/route.ts` | 优先使用前端已选标签，未传标签时才走自动匹配 |
