## 4. 关键阶段关卡

| 阶段过渡 | 继续条件 |
|----------|----------|
| Phase 0 → 1 | Charter 三方确认，资源到位 |
| Phase 1 → 2 | OCR 试行期指标全部达标；导出格式方案已确定 |
| Phase 2 → 3 | ≥5 名产研员完成完整闭环试用；无 P0 阻断 Bug |
| Phase 3 → 关闭 | 成熟期指标达标或趋近；1.0 验收报告签署 |

### 暂停条件

| 条件 | 处理方式 |
|------|----------|
| OCR 公式识别准确率连续两周无改善趋势 | 暂停精度优化，回到底层 OCR 引擎选型决策 |
| 系统在生产环境出现数据丢失或权限越界 | 暂停试运行，复现 + 修复后重新评估 |
| 产研侧反馈核心流程无法独立走通 | 暂停推广，退回优化 UX/培训 |
| OCR 单页处理超过 30 秒 | 需优化引擎或并行策略后方可继续量产导入 |
| 数据库查询在百题量级出现超时 | 需加索引或分页优化后方可支撑批量组卷 |

## 5. 后续版本边界

### 2.0 范围（初定）

- 智能组卷引擎（自动选题、难度均衡、知识点覆盖分析、题型自动配比、历史去重）
- 智能改编引擎（变体生成、数字替换、场景变换、难度升降级）
- 初高中理科覆盖
- Word / DOCX / 图片导入
- 几何图形图文分离
- 多 OCR 引擎降级切换

### 3.0+ 范围（初定）

- 学生端 / 家长端 / 教师端多角色终端
- 竞赛备考系统
- AI 助教（LLM 解题思路生成）

### 4.0 范围（初定）

- API 开放平台
- 对外服务
