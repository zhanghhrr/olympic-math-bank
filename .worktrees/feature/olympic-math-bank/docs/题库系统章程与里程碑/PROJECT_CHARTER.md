# 奥数题库管理系统 · 项目章程

> 版本：1.0 | 创建日期：2026-05-29 | 状态：已确认

---

## 1. 项目基本信息

| 字段 | 内容 |
|------|------|
| **项目名称** | 奥数题库管理系统（Olympic Math Bank） |
| **项目类型** | AI 题库 — 教研内容数字化与智能组卷系统 |
| **项目级别** | P1（战略项目） |
| **高风险项** | ① OCR 精度与公式识别 — LaTeX 还原错误影响教学正确性底线；② 数据存储与权限 — 教研题目为公司核心知识产权，需严格控制访问与存储安全 |
| **治理模式** | AI 辅助 + 人工审核：OCR 导入 → Draft → 人工编辑确认 → 提交审核 → Approved 入库。AI 做初稿，人类做最终决策 |

---

## 2. 需求与真实问题

| 字段 | 内容 |
|------|------|
| **表面需求** | 将公司历年积累的奥数纸质教材、试卷、练习册，以及已有电子版教研材料（PDF/Word），统一数字化归档为结构化题库，支持检索、管理与组卷 |
| **真实问题** | 教研资产分散在纸质文档与电子文件中，两者均存在同样困境：无法按知识点/年级/难度检索和复用；组卷依赖人工翻找、手动拼接，效率瓶颈严重；纸质材料 OCR 识别门槛高、电子版缺乏统一结构化标准，优质题目无法沉淀为可复用的数字化资产，新人无法快速调用历史积累 |
| **当前影响** | 教研员大量时间消耗在"找题 + 拼题"而非"教研设计"上；历史题目重复利用率低，每次考试/练习都需重新选题组卷；电子版素材散落在个人电脑/网盘，版本混乱，缺乏统一管理 |
| **受益方** | 小学教研员/产研员（直接用户）、授课教师（组卷出题）、学生（获得更高质量的练习与备考材料） |
| **不做代价** | 纸质 + 电子教研资产持续散落，无法规模化复用；新教研员学习成本高、无法利用历史沉淀；与数字化教研的行业趋势脱节，竞争壁垒无法建立 |

## 3. 项目目标

| 字段 | 内容 |
|------|------|
| **核心目标** | 建成从"材料导入 → OCR 识别 → 结构化入库 → 检索管理 → 智能组卷 → 排版打印"的完整奥数题库数字化闭环系统 |
| **首阶段目标** | ① 完成 OCR 自动化导入流水线，使纸质/电子 PDF 可高效转化为结构化题目；② 建立覆盖小学全模块的五级知识标签体系；③ 实现题目检索、编辑、审核、组卷打印的基础闭环；④ 课题组可独立使用系统完成从导入到打印的全流程 |
| **成功指标** | ① **可用性**：课题组能独立完成一轮完整的"材料导入 → 组卷打印"流程，无需技术人员介入；② **效率**：单份材料从原始 PDF 到可入库题目的处理时间相比人工录入显著降低；③ **规模**：题库积累到可支撑常规刷题课/考试组卷需求；④ **质量**：OCR 导入题目经人工修正后可达到教研使用标准 |

## 4. 项目范围

### 4.1 业务覆盖（按版本）

| 版本 | 业务范围 |
|------|----------|
| **1.0** | 小学数学全模块（P1-P6），覆盖小学产研团队 |
| **2.0** | 扩展至初高中理科（数学为主，物理/化学等理科场景） |
| **3.0** | 小初高全学科产研场景 |
| **4.0** | 开放平台 / 对外服务 |

### 4.2 输入材料覆盖（按版本）

| 版本 | 输入格式 |
|------|----------|
| **1.0** | PDF（纸质扫描 PDF + 电子导出 PDF） |
| **2.0** | 增加 Word / DOCX + 图片 |
| **3.0** | 增加 HTML、网页链接 |

### 4.3 1.0 版本承诺

- OCR 自动化导入流水线（PDF → 题目结构化 → 自动打标签 → 人工确认入库）
- 覆盖小学全模块的五级知识标签体系（1473 个标签节点）
- 题目 CRUD（四项题型）+ 版本历史 + 审核工作流
- 组卷打印（Notion-like 块状排版、学生/教师双模式）
- 三级权限体系（Admin / Reviewer / Editor）

### 4.4 1.0 版本不做

- 学生端 / 家长端
- 自动答题 / 智能批改
- 多 OCR 引擎自动降级切换
- 竞赛真题数据分析与组卷建议
- 几何图形题的图文分离
- 智能组卷（自动选题 + 难度均衡）
- 智能改编（变体生成）

### 4.5 远期愿景

- 智能组卷引擎（自动选题、难度均衡、知识点覆盖分析、历史去重）
- 智能改编引擎（基于已有题目生成变体）
- 多角色终端（学生端/教师端/家长端）
- API 开放平台
- AI 助教（LLM 解题思路生成）

## 5. 样本 / 数据 / 输入材料

| 字段 | 内容 |
|------|------|
| **首阶段样本量** | 约 10-20 份典型教研 PDF（涵盖不同年级、不同题型、不同排版复杂度），包括刷题课讲义、单元测试卷、竞赛集训资料等 |
| **材料类型** | PDF — 含纸质扫描件（图片型 PDF）和电子导出件（文字型 PDF）两类 |
| **覆盖的问题场景** | ① 纯文字题干（无公式）；② 含 LaTeX 公式的数学题；③ 含图形的几何题；④ 填空题（含下划线/横线占位）；⑤ 选择题（含选项排版）；⑥ 含答案与解析的教师版 |
| **样本归属** | 小学产研团队 |
| **验证方式** | 抽样比对：OCR 导入结果与原始 PDF 逐题人工校验，评估原文还原度、公式正确率、题型分类准确率、标签匹配准确率 |
| **排除的样本** | ① 纯手写材料（无印刷体）；② 严重破损/模糊的扫描件；③ 非 PDF 格式的原始文件（1.0 不做 Word/图片直接导入） |

## 6. 关键优化指标

### 6.1 质量指标

| 指标 | 试行期目标 | 成熟期目标 |
|------|-----------|-----------|
| **文字还原准确率**（OCR 文字部分与原文一致率） | ≥ 85% | ≥ 95% |
| **LaTeX 公式识别正确率**（公式可直接渲染，无需人工修正） | ≥ 70% | ≥ 90% |
| **题型分类准确率**（填空/选择/解答/计算 四类分对） | ≥ 90% | ≥ 98% |
| **知识标签匹配准确率**（Top-3 命中相关标签） | ≥ 75% | ≥ 90% |

### 6.2 效率与覆盖指标

| 指标 | 试行期目标 | 成熟期目标 |
|------|-----------|-----------|
| **单题人工修正量**（平均每道题需手动修改的字段数） | ≤ 3 | ≤ 1 |
| **产研员覆盖人数**（实际使用系统的产研员数） | ≥ 5 | ≥ 15 |

### 6.3 阶段关卡指标

| 指标 | 说明 |
|------|------|
| **核心流水线稳定性** | 连续 10 份 PDF 导入无阻断性错误 |
| **全流程可用性** | 课题组不依赖技术人员，独立走通"上传 → 编辑 → 入库 → 组卷 → 打印"闭环 |

## 7. 角色与决策权

### 7.1 管理角色

| 角色 | 职责 |
|------|------|
| **PMO / 项目管理负责人** | 管理里程碑进度、阶段关卡判定、资源协调 |
| **Project Owner（项目负责人）** | 产品方向决策、功能验收、范围变更审批 |
| **Business Owner / 业务对接人** | 代表小学产研团队，确认需求优先级、提供样本素材、组织试用与反馈收集 |

### 7.2 系统内角色

| 角色 | 权限 |
|------|------|
| **Admin** | 所有操作权限，含用户管理 |
| **Reviewer** | 可审核题目（Approved / Rejected） |
| **Editor** | 可创建/编辑/导入题目 |

### 7.3 决策权说明

- 范围变更（增减版本承诺、新增材料类型）：Project Owner 决策
- 样本质量判定与 OCR 结果验收标准：Business Owner + Project Owner 联合确认
- 技术方案选型（OCR 引擎、LLM 模型切换）：Project Owner 决策
- 阶段关卡「继续 / 暂停 / 缩减范围」：PMO 发起，Project Owner 终决

## 8. 阶段关卡与暂停条件

### 8.1 继续条件

| 阶段 | 继续条件 |
|------|----------|
| Phase 0 → 1 | Charter 三方确认，资源到位 |
| Phase 1 → 2 | OCR 试行期指标全部达标；导出格式方案已确定 |
| Phase 2 → 3 | ≥5 名产研员完成完整闭环试用；无 P0 阻断 Bug |
| Phase 3 → 关闭 | 成熟期指标达标或趋近；1.0 验收报告签署 |

### 8.2 暂停条件

| 条件 | 处理方式 |
|------|----------|
| OCR 公式识别准确率连续两周无改善趋势 | 暂停精度优化，回到底层 OCR 引擎选型决策 |
| 系统在生产环境出现数据丢失或权限越界 | 暂停试运行，复现 + 修复后重新评估 |
| 产研侧反馈核心流程无法独立走通 | 暂停推广，退回优化 UX/培训 |

### 8.3 返回样本 / 规则 / Prompt 的条件

| 场景 | 触发条件 |
|------|----------|
| **样本不足** | 当前 PDF 样本无法覆盖全部问题场景（如缺少几何图形题样本） |
| **规则冲突** | 知识标签体系出现层级归属争议，无法达成一致性判断标准 |
| **Prompt 退化** | LLM 标签匹配 Top-3 命中率持续低于 60%，需重新设计 Prompt 策略 |

### 8.4 性能 / 部署阻塞条件

| 阻塞项 | 说明 |
|--------|------|
| **OCR 单页处理超过 30 秒** | 量产导入时不可接受的时延，需优化引擎或并行策略 |
| **数据库查询在百题量级出现超时** | 需加索引或分页优化，否则无法支撑批量组卷 |
| **生产环境部署受阻** | Docker 编排 / 数据库迁移 / 密钥管理任一环节未就绪 |

### 8.5 超出当前版本范围 → 移入后续版本

| 请求 | 归入版本 |
|------|----------|
| Word / DOCX / 图片直接导入 | 2.0 |
| 智能组卷（自动选题 + 难度均衡） | 2.0 |
| 智能改编（变体生成） | 2.0 |
| 几何图形图文分离 | 2.0 |
| 学生端 / 家长端 | 3.0+ |
| API 开放平台 | 4.0+ |
