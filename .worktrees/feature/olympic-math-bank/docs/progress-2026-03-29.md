## 已完成的工作

### 1. OCR题目识别逻辑优化

**问题描述**:  
部分PDF中的题目格式为题号单独一行，内容在下一行，如：
```
6.
右下图是聪聪利用三角尺画出来的图形...
```

**修复方案**:  
在 `lib/ocr/question-identifier.ts` 中添加 `mergeQuestionNumbers` 方法：
- 使用正则表达式识别题号单独一行的模式
- 自动合并题号与下一行的题目内容
- 在内容处理的最开始阶段执行，确保后续分割逻辑正常工作

**代码变更**:
```typescript
private mergeQuestionNumbers(content: string): string {
  // 统一换行符
  let normalized = content.replace(/\r\n/g, '\n').replace(/\r/g, '\n');
  
  // 使用正则表达式合并题号和内容
  normalized = normalized.replace(
    /^(\s*\d+[\.．、])\s*\n(?!\s*\d+[\.．、]|\s*【|\s*!\[|\s*$)(.+)$/gm,
    '$1 $2'
  );
  
  return normalized;
}
```

**测试验证**:  
- 创建了测试脚本 `test-question-identify.ts`
- 验证了第6题和第8题（原单独一行题号）已正确合并
- 测试输出显示题号与内容已合并到同一行
